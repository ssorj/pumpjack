Module client

An imperative client API.

Blocking

The endpoint-lifecycle methods connect(), open-session(), open-sender(), and open-receiver() start their respective operations but do not complete them. Use <endpoint>.wait() to block until a connect or open operation completes. <endpoint>.close() returns an awaitable task.

Sender send() blocks until there is credit to send the message. It does not wait until the message is acknowledged. The send() method returns an awaitable task that resolves to a tracker for a completed delivery.

Receiver next() is designed to work with automatic credit management (which is enabled by default). It blocks until a message is available to return. Receiver receive(), by contrast, is for explicit credit management. It issues a single credit and returns a task representing the completed delivery.

send() and next() have non-blocking variants called try-send() and try-next(). The former returns null if there is no credit to send. The latter returns null if there is no delivery already present.

Blocking operations take optional timeout arguments. If the timeout is exceeded, they raise a timeout error.

Core model

ClassSummary
client A top-level container of connections, sessions, senders, and receivers
connection A channel for communication between two peers on a network
session A container of senders and receivers
sender A channel for sending messages
receiver A channel for receiving messages
